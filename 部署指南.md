# 🚀 个人网站部署指南

本指南将教你如何将个人作品集网站免费部署上线。

---

## 📋 部署前检查清单

在部署之前，请确保：

- [ ] 所有图片已添加到 `assets/` 文件夹
- [ ] 个人信息已更新（邮箱、联系方式等）
- [ ] 在本地浏览器测试过，确认显示正常
- [ ] 有GitHub账号（https://github.com/YitongPeng）

---

## 🎯 方案一：GitHub Pages（推荐）

### 优点
- ✅ 完全免费
- ✅ 自动部署
- ✅ 支持自定义域名
- ✅ 稳定可靠

### 部署步骤

#### 步骤1：初始化Git仓库

```bash
# 进入项目文件夹
cd /Users/pengyitong/Documents/Project/portfolio

# 初始化Git仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "初始化彭弋桐Riley的个人作品集网站"
```

#### 步骤2：创建GitHub仓库

1. 打开浏览器，访问：https://github.com/new
2. 填写仓库信息：
   - **Repository name**：`portfolio`（或任何你喜欢的名字）
   - **Description**：个人作品集网站
   - **Public**（公开）- 必须选择Public才能使用免费的GitHub Pages
   - ❌ **不要**勾选 "Add a README file"（我们已经有了）
3. 点击 **Create repository**

#### 步骤3：推送到GitHub

复制GitHub上显示的命令（类似下面这样）：

```bash
# 添加远程仓库
git remote add origin https://github.com/YitongPeng/portfolio.git

# 推送到GitHub
git branch -M main
git push -u origin main
```

> 💡 **提示**：如果遇到需要登录，使用你的GitHub用户名和密码（或个人访问令牌）

#### 步骤4：启用GitHub Pages

1. 在GitHub仓库页面，点击 **Settings**（设置）
2. 左侧菜单找到 **Pages**
3. 在 **Source** 下拉菜单中：
   - Branch: 选择 `main`
   - Folder: 选择 `/ (root)`
4. 点击 **Save**
5. 等待1-2分钟，页面会显示你的网站地址

🎉 **完成！** 你的网站地址是：`https://yitongpeng.github.io/portfolio/`

#### 步骤5：访问你的网站

在浏览器中输入：`https://yitongpeng.github.io/portfolio/`

> 💡 **首次部署可能需要5-10分钟才能生效，请耐心等待**

## 📝 方案二：Vercel（更快速）

### 优点
- ✅ 免费
- ✅ 部署速度更快
- ✅ 自动HTTPS
- ✅ 全球CDN加速

### 部署步骤

#### 步骤1：推送到GitHub
（与方案一的步骤1-3相同）

#### 步骤2：导入到Vercel

1. 访问：https://vercel.com
2. 使用GitHub账号登录
3. 点击 **Add New** → **Project**
4. 选择 `portfolio` 仓库
5. 点击 **Deploy**

🎉 **完成！** Vercel会自动分配一个域名，如：`portfolio-xxx.vercel.app`

## 🌐 方案三：Netlify（备选）

### 优点
- ✅ 免费
- ✅ 简单易用
- ✅ 表单功能（如果需要联系表单）

### 部署步骤

#### 方式A：通过GitHub

1. 访问：https://www.netlify.com
2. 使用GitHub账号登录
3. 点击 **Add new site** → **Import an existing project**
4. 选择 **GitHub** → 授权 → 选择 `portfolio` 仓库
5. 点击 **Deploy site**

#### 方式B：拖拽部署（最简单）

1. 访问：https://app.netlify.com/drop
2. 直接将 `portfolio` 文件夹拖拽到页面
3. 自动部署完成

🎉 **完成！** 网站地址类似：`your-site-xxx.netlify.app`

## 🔄 如何更新网站？

### 方法1：通过Git推送（推荐）

```bash
# 修改文件后
git add .
git commit -m "更新个人信息"
git push origin main
```

**GitHub Pages / Vercel / Netlify 会自动检测到更新并重新部署！**

### 方法2：直接在GitHub网页上编辑

1. 进入GitHub仓库
2. 找到要修改的文件（如 `index.html`）
3. 点击编辑按钮（铅笔图标）
4. 修改后点击 **Commit changes**
5. 等待自动部署完成

---

## 🎨 可选：绑定自定义域名

### 如果你有自己的域名（如 rileyp eng.com）

#### 在GitHub Pages中绑定

1. 在仓库的 **Settings** → **Pages**
2. 在 **Custom domain** 输入你的域名
3. 在域名提供商（如阿里云、GoDaddy）添加DNS记录：
   ```
   类型: CNAME
   主机记录: www
   记录值: yitongpeng.github.io
   ```
4. 等待DNS生效（可能需要几小时）

#### 在Vercel中绑定

1. 进入项目的 **Settings** → **Domains**
2. 添加你的域名
3. 按照提示添加DNS记录
4. Vercel会自动配置HTTPS

## ⚡ 性能优化建议

### 1. 压缩图片

推荐工具：
- **TinyPNG**: https://tinypng.com/
- **Squoosh**: https://squoosh.app/

将 `assets/` 文件夹中的图片压缩后再上传，可以大幅提升加载速度。

GitHub Pages和Vercel/Netlify都会自动处理缓存，无需额外配置。

## 🐛 常见问题

### Q: 部署后图片不显示？

**A**: 检查图片路径是否正确：
```html
<!-- 正确 -->
<img src="assets/个人简历照.png">

<!-- 错误（多了斜杠） -->
<img src="/assets/个人简历照.png">
```

### Q: CSS样式没有生效？

**A**: 确保 `style.css` 和 `index.html` 在同一目录，且路径正确：
```html
<link rel="stylesheet" href="style.css">
```

### Q: GitHub Pages显示404？

**A**: 
1. 确认仓库是Public（公开）
2. 检查Settings → Pages是否正确配置
3. 等待5-10分钟让部署完成

### Q: 修改后网站没有更新？

**A**: 
1. 清除浏览器缓存（Ctrl+F5 / Cmd+Shift+R）
2. 等待3-5分钟让CDN缓存刷新
3. 检查GitHub Actions是否部署成功
